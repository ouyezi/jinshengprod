# 研发晋升评审系统 — 使用简介

> 系统地址：http://10.55.10.188:5173/
> 环境：内部测试环境
> 请使用 Chrome / Edge / Safari 等现代浏览器访问

---

## 一、系统定位

本系统用于支撑研发团队职级晋升评审工作，覆盖：

- **管理端**：员工信息维护、晋升标准配置、评审结果汇总（需管理员登录）
- **评委端**：匿名打分、系统生成评审建议、结果提交（无需登录，直接访问）

---

## 二、管理端操作说明

### 2.1 登录

访问：**http://10.55.10.188:5173/admin/login**

| 字段 | 内容 |
|------|------|
| 用户名 | `admin` |
| 密码 | `dongfu123` |

登录后自动跳转至管理后台首页。

### 2.2 员工管理（/admin/employees）

用于维护参评员工信息：

- **新增员工**：手动录入姓名、当前职级、目标职级
- **批量导入**：下载 Excel 模板 → 按模板填写 → 上传导入
- **编辑/删除**：对已有员工信息进行调整
- **导出**：支持导出员工名单

> 员工是评审的最小单位，评委端搜索打分均基于此处维护的人员数据。

### 2.3 晋升标准配置（/admin/standards）

按职级（P5 ~ P10）维护 **12 项能力维度** 的评审标准描述：

| 维度 | 说明 |
|------|------|
| 务实 | 工作作风与实际问题解决 |
| 担当 | 责任感与主动性 |
| 追求卓越 | 自我驱动与高标准要求 |
| 学习创新与效率提升 | 技术/方法创新能力 |
| 技术专业与质量 | 代码质量与技术深度 |
| 架构能力 | 系统设计能力 |
| 业务理解能力 | 对业务领域的掌握程度 |
| 执行力 | 任务推进与落地能力 |
| 团队协作 | 跨团队沟通配合 |
| 知识传承与影响力 | 技术布道、带人能力 |
| 基础工作产出 | 日常工作交付量与质量 |
| AI 使用深度 | AI 工具应用水平 |

> 每项标准按 P5~P10 分级描述，评委端打分时可直接对照。

### 2.4 评审汇总（/admin/summary）

评审结束后，管理员可在此查看和导出全部结果：

- **筛选条件**：按员工姓名、评委姓名过滤
- **明细展示**：26 列完整数据（员工信息 + 12 项得分 + 平均分 + 系统建议 + 评委结论 + 优势/不足）
- **Excel 导出**：一键导出全部汇总数据，用于归档或线下讨论

## 三、评委端操作说明

### 3.1 访问方式

访问：**http://10.55.10.188:5173/review**

> 无需登录，直接打开即可使用。

### 3.2 打分流程

1. **搜索员工**：输入姓名或工号，找到待评审对象
2. **查看标准**：页面自动展示该员工目标职级对应的 12 项能力标准
3. **逐项打分**：每项 1~5 分，支持随时保存草稿
4. **生成结果**：点击「生成结果」，系统自动计算：
   - 价值观平均分（前 3 项）
   - 能力平均分（中间 8 项）
   - 产出平均分（后 1 项）
   - 综合得分及晋升建议
5. **填写评价**：补充「优势」「不足」文字描述
6. **提交结果**：确认无误后提交，提交后不可修改

### 3.3 状态说明

| 状态 | 含义 |
|------|------|
| 待提交 | 已保存草稿，尚未提交 |
| 已提交 | 评审结果已确认提交 |

## 五、注意事项

1. **当前为测试环境**，数据会随服务重启保留（SQLite 持久化），但请勿存放敏感生产数据。
2. **评委端匿名**：评委无需注册/登录，填写姓名即可打分，请评审负责人自行登记评委身份与对应记录。
3. **浏览器兼容**：请使用 Chrome 90+ / Edge 90+ / Safari 14+。
4. **并发限制**：当前单进程部署，建议同一时间评委人数不超过 20 人。
